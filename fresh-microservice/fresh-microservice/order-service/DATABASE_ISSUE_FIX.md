# OrderItem Field 'price_at_purchase' Database Issue - FIX DOCUMENTATION

## Problem Summary
The application was throwing the following error when trying to insert OrderItems:

```
java.sql.SQLException: Field 'price_at_purchase' doesn't have a default value
```

## Root Cause
In the `OrderItem` entity, the `priceAtPurchase` field was marked as `nullable = true` in the JPA mapping:

```java
@Column(name = "price_at_purchase", nullable = true, precision = 10, scale = 2)
private BigDecimal priceAtPurchase;
```

However, the database column was created as `NOT NULL` without a default value, causing a mismatch. When Hibernate tried to insert a new `OrderItem` without providing a value for this field, MySQL rejected it.

## Solution Applied

Updated the column definition to explicitly allow NULL values with a default:

```java
@Column(name = "price_at_purchase", nullable = true, precision = 10, scale = 2, columnDefinition = "DECIMAL(10, 2) DEFAULT NULL")
private BigDecimal priceAtPurchase;
```

### What This Does:
- `nullable = true` - Tells Hibernate that this field can be null
- `columnDefinition = "DECIMAL(10, 2) DEFAULT NULL"` - Forces Hibernate to create the database column with an explicit NULL default value
- `precision = 10, scale = 2` - Maintains the numeric precision (e.g., 9999.99)

## How to Prevent This Issue in the Future

### Best Practice 1: Use `columnDefinition` for Optional Columns
Always explicitly define the column definition for optional fields:

```java
@Column(name = "optional_field", nullable = true, columnDefinition = "DECIMAL(10, 2) DEFAULT NULL")
private BigDecimal optionalField;
```

### Best Practice 2: Set Default Values in Entity
Initialize nullable fields with default values in the entity:

```java
@Column(name = "price_at_purchase", nullable = true, precision = 10, scale = 2)
private BigDecimal priceAtPurchase = null;
```

### Best Practice 3: Use @Generated Annotation
For fields that should be generated by the database:

```java
@Generated(GenerationTime.INSERT)
@Column(name = "created_timestamp", nullable = false)
private LocalDateTime createdTimestamp;
```

### Best Practice 4: Pre-persistence Initialization
Initialize nullable fields in a `@PrePersist` method:

```java
@PrePersist
public void initializeDefaults() {
    if (priceAtPurchase == null) {
        priceAtPurchase = BigDecimal.ZERO;  // or leave as null if truly optional
    }
}
```
